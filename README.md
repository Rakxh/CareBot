# CareBot: AI-Powered Health Assistant

A RAG-based conversational AI system for health information, using LangChain and FAISS for retrieval, an agentic workflow that can call OpenAI GPT or Google Gemini, and a Streamlit chat interface. The vector index can be backed up to and restored from Google Drive for zero-downtime availability.

## Tech Stack

Python, LangChain, OpenAI API, FAISS, Google Gemini API, Streamlit, Google Drive API

## Dataset

`data/medical_documents/` contains 20 sample health reference documents (one per condition: diabetes, hypertension, asthma, allergies, migraine, and others), each structured with overview, symptoms, causes, treatment, and prevention sections. They were generated by `src/generate_sample_documents.py` and represent a small seed set; `src/ingestion.py` is built with `DirectoryLoader` so it scales to a much larger corpus (1000+ documents) without code changes.

## How It Works

`src/ingestion.py` loads documents and splits them into overlapping chunks with `RecursiveCharacterTextSplitter`. `src/embeddings.py` and `src/llm_router.py` let you swap between OpenAI and Gemini for embeddings and chat completion respectively, controlled by the `EMBEDDING_PROVIDER` and `LLM_PROVIDER` environment variables. `src/vector_store.py` builds and persists a FAISS index from the chunks. `src/agent.py` wraps the FAISS retriever as a LangChain tool and builds a tool-calling agent that decides when to search the knowledge base before answering. `src/drive_sync.py` uploads or downloads the FAISS index folder to/from a Google Drive folder using a service account, so a fresh deployment can restore the index without rebuilding it from scratch.

## Setup

```bash
pip install -r requirements.txt
cp .env.example .env
```

Fill in `.env` with your OpenAI and/or Google Gemini API keys. For Google Drive sync, create a service account, share the target Drive folder with its email address, download its JSON key, and point `GOOGLE_DRIVE_CREDENTIALS` at the file.

## Build the Vector Index

```bash
python src/generate_sample_documents.py
python src/vector_store.py
```

This ingests the documents in `data/medical_documents/`, embeds them, and saves a FAISS index to `vector_db/`.

## Run the App

```bash
streamlit run app/streamlit_app.py
```

Use the sidebar to choose an LLM/embedding provider, sync the index from Google Drive, or rebuild it from the local documents.

## Run with Docker

```bash
docker build -t carebot-health-assistant .
docker run -p 8501:8501 --env-file .env carebot-health-assistant
```

## Run Tests

```bash
pytest tests/
```

Tests cover the offline document ingestion and chunking pipeline; they do not call any external LLM or embedding API.
